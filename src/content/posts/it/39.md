Astro è un framework moderno pensato per la realizzazione di siti statici e ibridi ad alte prestazioni. Nato nel 2021, ha rapidamente guadagnato terreno grazie al suo approccio server-first e a un'architettura a isole che riduce drasticamente il peso del JavaScript distribuito al browser. Questo modello consente di ottenere pagine rapide, ottimizzate per la SEO e facili da mantenere nel tempo, rovesciando la logica delle Single Page Application in cui tutto il rendering avviene lato client.

A differenza delle SPA che idratano l'intera interfaccia nel browser, Astro permette un'idratazione selettiva: solo i componenti interattivi vengono trasformati in JavaScript e resi attivi, mentre il resto della pagina resta statico. Questo concetto, noto come partial hydration, si basa sulle cosiddette isole, ovvero porzioni isolate di interfaccia che ricevono JavaScript soltanto quando necessario. La direttiva `client:visible`, per esempio, ritarda il caricamento di un componente finché non entra nel viewport, riducendo il lavoro che il browser deve svolgere al primo caricamento della pagina.

```javascript
// Esempio di isola
import Hero from '../components/Hero.astro';

<main>
  <Hero client:visible />
</main>;
```

Ogni componente può essere costruito in Astro oppure integrato da framework come React, Vue o Svelte. Questa compatibilità consente di riutilizzare codice esistente o di migrare progressivamente, senza dover ricostruire tutto da zero. La scelta del framework è per componente, non per progetto, il che significa che una stessa pagina può contenere un elemento React accanto a uno Svelte, con Astro che si occupa di coordinarne il ciclo di vita e di consegnare al browser solo il codice effettivamente necessario per ciascuno di essi.

##### Struttura del progetto e routing

Dal punto di vista dello sviluppatore, Astro adotta una sintassi simile a HTML ma con alcune estensioni. I file `.astro` possono contenere script JavaScript racchiusi tra delimitatori `---`, e offrono lo scoping automatico per i CSS, garantendo che gli stili di un componente non fuoriescano verso il resto della pagina. La struttura del progetto è semplice e ordinata, con cartelle dedicate ai componenti riutilizzabili, ai layout e alle pagine che corrispondono alle route pubbliche dell'applicazione.

```javascript
src/
├── components/
├── layouts/
└── pages/
```

Ogni file dentro `pages/` diventa automaticamente una route. Per gestire contenuti dinamici si utilizzano file con nomi parametrizzati, dove la parte tra parentesi quadre indica la variabile che verrà estratta dall'URL. All'interno del componente, il parametro è accessibile attraverso l'oggetto `Astro.params`, rendendo il routing espressivo e prevedibile senza richiedere configurazioni aggiuntive.

```javascript
// src/pages/blog/[slug].astro
const { slug } = Astro.params;

<h1>Post: {slug}</h1>;
```

##### Rendering e layout

Il meccanismo di rendering è altamente flessibile. Di default, Astro utilizza lo Static Site Generation, ma è possibile abilitare il rendering lato server o una modalità ibrida che combina pagine statiche e dinamiche nello stesso progetto. La configurazione avviene nel file `astro.config.mjs` con una singola proprietà, e ogni pagina può sovrascrivere il comportamento globale con una direttiva locale, permettendo di gestire casi come dashboard autenticate accanto a pagine di contenuto completamente statiche.

```javascript
import { defineConfig } from 'astro/config';

export default defineConfig({
  output: 'server', // oppure 'static' | 'hybrid'
});
```

I layout funzionano come template riutilizzabili che possono includere header, footer e slot dove iniettare contenuti dinamici. Questo sistema di composizione consente di mantenere la struttura visiva coerente tra le pagine senza duplicare il markup, e il passaggio delle proprietà avviene attraverso `Astro.props` con supporto completo ai tipi TypeScript.

##### Contenuti strutturati e ottimizzazione delle risorse

Astro è particolarmente indicato per siti orientati ai contenuti. L'integrazione con Markdown e la possibilità di definire collezioni tramite TypeScript consente di trattare contenuti statici come dati strutturati, con validazione automatica degli schemi a tempo di build. Se un campo obbligatorio manca nel frontmatter di un articolo, Astro blocca la build e segnala l'errore con precisione, evitando che dati malformati raggiungano la produzione.

```javascript
// src/content/config.ts
import { z, defineCollection } from 'astro:content';

const blog = defineCollection({
  schema: z.object({
    title: z.string(),
    publishDate: z.date(),
    tags: z.array(z.string()),
  }),
});

export const collections = { blog };
```

Per la gestione delle immagini, Astro mette a disposizione strumenti di ottimizzazione automatica che generano formati moderni come WebP e AVIF, ridimensionano le immagini per i diversi breakpoint e aggiungono i metadati di larghezza e altezza necessari a prevenire il layout shift. Il supporto per il data fetching è altrettanto completo: è possibile recuperare dati da API REST o GraphQL sia durante la build che in modalità SSR, con una sintassi asincrona che si integra naturalmente nella sezione di script del componente.

##### Evoluzione e prospettive

Astro continua ad evolversi in modo deciso. Con la versione 4 sono state introdotte funzionalità sperimentali per il prerendering predittivo tramite Speculation Rules API, che consente al browser di precaricare le pagine prima ancora che l'utente faccia click su un link, e nuove opzioni per la gestione dei percorsi dinamici. L'Astro Dev Toolbar, integrata nell'interfaccia di sviluppo, offre ispezione in tempo reale delle isole attive, dei tempi di idratazione e delle violazioni delle regole di accessibilità, portando una visibilità sul comportamento del framework che in altri strumenti richiede plugin esterni.

Astro si rivela una soluzione potente e moderna per chi desidera costruire siti veloci, modulari e scalabili. Il suo ecosistema flessibile, la compatibilità con più framework e la capacità di gestire contenuti in modo strutturato lo rendono uno strumento particolarmente efficace per blog, documentazioni, landing page e portali editoriali dove la velocità di caricamento e la qualità del markup influiscono direttamente sulla visibilità nei motori di ricerca e sull'esperienza dell'utente finale.
